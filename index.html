Everyday-Git

What is GIT?

Git is a free and open source distributed version control system(VCS) designed to handle everything from small to very large projects with speed and efficiency.

Now you might be asking yourself what a version control system is?

A Version Control System is simply a system that records changes to a file or set of files over time so that you can recall specific versions of the file later.

Other version control systems include :

  • Subversion (SVN)
  • Mercurial
  • CVS
  • Perforce

Installing GIT

To check if Git is installed on your machine, open your terminal and type:

git --version

If Git is installed, you will see the version number e.g in this case 2.49.0

If Git is not installed, you will see a message indicating that the command is not recognized.

To install Git, follow these steps:

  1. Go to the official Git Downloads web page

  2. Download the installer for your operating system. eg (Windows,Mac,Linux)
  3. Run the installer and follow the instructions just as prompted, you can just use defaults for now
  4. Once installed, verify the installation by running git --version in your terminal.
  5. If you see a version, you are ready to proceed
  6. If you see a message indicating that the command is not recognized, restart your terminal and try again.
  7. If all else fails, identify any errors, look them up or try a video tutorial

Configuring Git

To configure Git, you need to set your username and email address. This information will be associated with your commits.

Run the following commands in your terminal:

git config --global user.name "your name"

Recommended: Replace "your name" with your actual github username.

git config --global user.email "your emaiil"

Recommended: Replace "your email" with your actual github email.

To verify your configuration, run the following command:

git config --list

This will display your Git configuration settings, including your username and email address.

Creating a Git Repository

A Git repository is a directory that contains all the files and folders for your project, along with the version history of those files.

To create a new Git repository, navigate to the directory where you want to create the repository and run the following command:

git init

This command initializes a new Git repository in the current directory.

After running this command, you will see a new hidden directory called .git in your project folder. This directory contains all the information about your Git repository, including the version history and configuration settings.

If you don't see it , no worries , it could be that your OS is set to keep the files hidden

To check the status of your Git repository, run the following command:

git status

This command will show you the current status of your repository, including any changes that have been made to the files.

Adding files to the staging area

The staging area is a temporary area where you can prepare files before committing them to the repository. When you add files to the staging area, Git tracks the changes made to those files.

To add files to the staging area, use the git add command followed by the file name:

git add filename

For example, to add a file called example.txt, run the following command:

git add example.txt

You can also add all files in the current directory to the staging area by using the . (dot) symbol:

git add .

This command adds all changes in the current directory to the staging area.

To check the status of the staging area, run the following command:

git status

This command will show you the current status of your repository, including any changes that have been made to the files.

In the output, you will see the files that are staged for commit and those that are not staged. Staged files are ready to be committed, while unstaged files are not.

Tracked files are files that Git is already tracking, while untracked files are new files that Git has not yet started tracking. You can add untracked files to the staging area using the git add command.

Committing changes

To commit changes to your Git repository, use the git commit command followed by the -m option and a commit message:

git commit -m "Your commit message"

For example, to commit changes with the message "Add new feature", run the following command:

git commit -m "Add new feature"

It's important to write clear and descriptive commit messages that explain the changes made in the commit. This helps you and others understand the history of the project.

The basic rule of thumb is your commit message should answer this question

This commit will ....

For example: For our case

git commit -m "add example.txt and main.txt"

To view past commits:

git log

This command will display a list of all commits made in the repository, along with their commit messages, author information, and timestamps.

I added a new commit for the purposes next few steps, effectively adding a new.txt file

To view a shorter and concise history, use:

git log --oneline

This command will display a condensed version of the commit history, showing only the commit hash and the commit message.

To go back to a specific commit, you can use the git checkout command followed by the commit hash:

git checkout commit_hash

To find the commit hash, you can use the git log command to view the commit history and copy the hash of the desired commit.

Once you run the git checkout command, your working directory will be updated to reflect the state of the repository at that specific commit.

For example, to go back to a commit with the hash abc123, run the following command:

git checkout abc123

Replace abc123 with the actual commit hash you want to go back to.

eg: in our case, 04083d5

git checkout 04083d5

This command will switch your working directory to the state of the repository at that specific commit.

Branches

Branches are a powerful feature of Git that allows you to work on different versions of your project simultaneously. A branch is essentially a separate line of development that diverges from the main branch (usually called main or master).

Creating a Branch

To create a new branch, use the git branch command followed by the name of the new branch:

git branch new_branch_name

For example, to create a new branch called feature-branch, run the following command:

git branch feature-branch

This command creates a new branch called feature-branch based on the current branch.

Viewing all branches

To view your current branch and all branches in your repository, use the git branch command without any arguments:

git branch

This command will display a list of all branches in your repository, with an asterisk (*) next to the current branch.

Switching active branches

To switch to the new branch, use the git switch command followed by the branch name:

git switch feature-branch

This command switches your working directory to the feature-branch branch.

To verify that you have switched to the new branch, run the git branch command again:

git branch

This command will show you the current branch with an asterisk (*) next to it.

To create and switch to a new branch, there is a shortcut

git switch -b new_branch_name

For example, to create and switch to a new branch called feature-branch, run the following command:

git switch -b feature-branch

This command creates a new branch called feature-branch and switches your working directory to that branch.

Merging a branch

For this exercise , we are going to go into feature-branch and add a feature/make changes

To do this, we will create a new file called feature.txt

type nul > feature.txt

Then we will add it to the staging area

git add feature.txt

Then we will commit the changes

git commit -m "add feature.txt"

Now we are ready to merge the branch into the master branch

To merge a branch into the current branch, first switch to the branch you want to merge into (usually the main branch) using the git switch command:

git switch master

Then, use the git merge command followed by the name of the branch you want to merge:

git merge feature-branch

This command merges the changes from the feature-branch into the current branch (in this case, the main branch).

This command will merge the changes from the feature-branch into the main branch.

Deleting a branch

After merging, you can delete the branch if you no longer need it using the git branch -d command followed by the branch name:

git branch -d feature-branch

This command deletes the feature-branch branch from your repository.
